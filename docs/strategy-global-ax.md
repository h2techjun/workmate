# Workmate 글로벌·AX 전략 재설계 (2026-07-03)

> 상위 문서: [`monetization-strategy.md`](./monetization-strategy.md)(수익 3-Track)를 대체하지 않고 **확장**한다.
> 이 문서는 "어디서 트래픽을 끌어오고, AI 전환(AX) 시대에 무엇으로 살아남는가"를 정의한다.

## 0. 한 줄 재정의

> **"한국 실무 계산기 모음" → 세계인이 한국을 만나는 모든 접점의 숫자·규정 도구(Korea OS),
> 그리고 AX 시대에 AI가 인용(cite)하고 호출(call)하는 한국 표준 데이터원.**

우리의 자산은 계산기 UI가 아니라 **"출처·기준일이 달린, 유지보수되는 한국 규정·수치의 순수함수 라이브러리"**다.
UI(웹)는 그 자산의 첫 번째 유통 채널일 뿐이며, AX 시대에는 AI 답변 인용·에이전트 API가 두 번째, 세 번째 채널이 된다.

## 1. 진단 — 왜 지금 재설계인가

### 실측 (GSC 3개월)
- 클릭 42 / 노출 3.52k / 평균순위 12.9. **미국(9.2위) > 한국(14.1위)** — 영어 니치가 검증된 레인.
- 승자 패턴 = **school-grade** (영어 × 한국 고유 로직 × 계산 필요): 20클릭/531노출/5.8위.
- 패자 패턴 = 무상태 변환기(percent·distance·pyeong): 구글 인라인 답변이 흡수 → CTR 구조적 저하.

### 구조적 위협 (AX)
검색 자체가 AI 답변(AI Overviews·ChatGPT·Perplexity)으로 이동 중이다. **단순 변환·단답 정보는 AI가 페이지 방문 없이 답한다.**
→ 무상태 변환기 중심의 트래픽 모델은 중기적으로 침식된다. 이미 우리 CTR 데이터에 나타나 있다.

### 구조적 기회 (AX를 뒤집기)
AI는 한국 특수 규정·수치(전세 제도, 만나이, 비자, 세율, 4대보험)에 대해 **신뢰할 수 있는, 날짜 있는, 출처 있는 근거**가 필요하다.
우리의 기존 도그마 — 순수함수 + KS/법령 출처 + 기준일 + 면책 — 는 **우연히도 GEO(Generative Engine Optimization)에 최적화된 체질**이다.
부족한 것은 체질이 아니라 **노출 방식**(llms.txt, 데이터 신선도 공개, API)이다.

### 결론
1. AI가 대체 못 하는 것에 집중: **다입력·상태·규정 결합 도구**(deposit-risk, 세금, 비자 자격) + **유지보수되는 최신 수치**.
2. AI에게 대체당하지 말고 **AI의 출처가 된다**: 인용 최적화 + 에이전트 호출 가능화.
3. 트래픽 풀 자체를 넓힌다: 영어 다음 언어, 거주자 다음 여정 단계.

## 3. 전략 6축

### P1. 여정 단계 커버리지 (Before → In → After Korea)
현재 도구 71개는 "In Korea"에 집중. 여정 전체로 확장하되 **도구-우선**(글-only 금지):
- **Before**: cost-of-living(완), 비자 체커(D-8·F-2 완 → E-7·D-10 후보), 유학 예산, 초기 정착비(보증금+가구+수수료 일회성) 계산기
- **In**: 주거(깡통전세 완)·세금·보험·송금(완) — 심화 유지
- **After**: pension-refund(완), 출국 정산 체크리스트(pension-refund 가이드에 이미 존재 → 도구화 검토)
- **Ring 3용 경량 도구**: korean-age의 팬덤 확장("아이돌 만나이" — 팬 검색 실존 패턴), 한국 금액 읽기(만/억 — K-드라마·부동산 뉴스 맥락), 콘서트·여행 예산. school-grade 승자 공식의 복제.

### P2. 다국어 — 측정 기반 단계 확장 (전량 번역 금지)
- **1차 파일럿: 베트남어(vi)** — 근거: 위 통계 + 콘텐츠 공백 + 한국 송금 핀테크(GME·Hanpass·E9pay 등)의 최우선 마케팅 타깃 = 제휴 정합.
- **대상**: 전체 71개가 아니라 **외국인 고통 TOP 8**만 (deposit-risk, jeonse-wolse, cost-of-living, remittance, korean-age, visa-days, foreign-health-insurance, pension-refund + net-salary 검토).
- **2차 후보**: zh(유학생 7.3만 타깃), th/uz/ne/km(E-9), ja(Ring 3 관광·팬덤, RPM 높음) — **GSC 국가별 데이터로 검증 후** 확장.
- **품질 게이트**: 기계번역 초안 + 용어집(전세=jeonse 병기 등) + 숫자 중심 도구부터. ToolGuide 번역 없이는 출시 금지(AdSense thin 규칙 동일 적용).
- next-intl 로케일 라우팅은 이미 구조 지원 — 기술 부채 낮음.

### P3. AX 아키텍처 — 인용되고 호출되는 사이트
- **GEO 기본기**: `llms.txt`(+`llms-full.txt`), 기존 FAQPage/HowTo JSON-LD 유지, 수치 테이블의 깨끗한 HTML 표화.
- **`/data` 신선도 레지스트리 (핵심·저비용·고레버리지)**: `lib/constants/`의 모든 변동 수치(기준금리, 최저임금, NHIS 요율, 세율표, 전월세전환율…)를 "값 + 근거 법령/고시 + 최종 검증일 + 변경 이력"으로 공개하는 단일 페이지. 효과 ① AI 인용 최적 형식 ② 사용자 신뢰(E-E-A-T) ③ 내부 업데이트 규율 강제(audit 연동) ④ 링커블 에셋.
- **Korea Standards API / MCP 서버**: `lib/calculations/`는 이미 순수·타입·테스트 완비 → route handler + MCP manifest로 노출 비용이 극히 낮다. "한국 이주를 계획하는 AI 에이전트가 호출하는 계산 엔드포인트" 포지션 선점. 단기 수익보다 **개발자 커뮤니티 백링크·PR·선점 효과**. (남용 방지 rate limit, 무료 시작.)
- **AI 유입 측정**: referral/UA 기준 perplexity·chatgpt·copilot 유입 분리 트래킹 → AX 채널의 실효를 데이터로 판단.

### P4. 유통 엔진 — SEO 단일 의존 탈피 (도메인 권위가 병목: 평균 12.9위)
- **커뮤니티**: Reddit(r/Living_in_Korea 등)은 영어권, **베트남·필리핀 커뮤니티는 Facebook 그룹이 주 채널** — vi 출시와 동시 시딩.
- **임베드 위젯**: 계산기 iframe 임베드(출처 링크 필수) — Wise 플레이북. 블로거·유학원·행정사 사이트가 백링크 공급원이 된다.
- **링커블 에셋**: 연 1회 "외국인 한국 생활비 리포트" / "깡통전세 위험 지표" — 기자·블로거가 인용할 데이터 자산.
