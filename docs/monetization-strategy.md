# Workmate 수익화 전략 (정직한 로드맵)

> 목적: "AdSense로만 환전"하는 함정을 피하고, 이미 만든 자산(도구 68개 + SEO + UX)을
> 실제 수익으로 바꾸는 단일 진실원.
> 최종 갱신: 2026-06-29. 근거: GSC 실적 + 검증된 제휴망 조사 + AdSense 공식 정책.

---

## 0. 핵심 공식 — 수익 = 트래픽 × 방문자당 단가

현재 실적(GSC, 최근 28일):
- 총 클릭 **39** (≈ 하루 1.4명 검색 유입) · 노출 **3,040** · CTR **1.3%** · 평균 순위 **12.4**

숫자가 말하는 것:
- 평균 순위 12.4 = **구글 2페이지 문턱.** 1페이지(10위 내) 진입 시 CTR 1.3% → 5~10%로 뛰며 클릭이 수 배.
- 노출이 월초 ~20/일 → 월말 ~150/일로 **한 달 만에 7배 성장.** 신규 사이트 궤도로는 정상.
- **이 트래픽으로는 AdSense 단독 기대수익 월 수백 원.** AdSense는 트래픽 장사다.

→ **결론: 방향(도구+SEO)은 옳고 자산은 잘 쌓인다. 문제는 "AdSense 단독"이라는 환전 방식과, 공통 병목인 트래픽이다.**

## 1. 3-Track 수익 모델 (순차 활성화)

### Track 1 — AdSense (바닥 수익, 지금)
- 승인 후 결과 하단 광고. 한국어 계산기 RPM은 낮다. **유지하되 주력 아님.**
- **영어(/en) 페이지 RPM이 한국어의 3~5배** (USD 광고주·낮은 경쟁). → 영어 트래픽을 우선하면 "적은 트래픽 × 높은 단가"가 성립.
- 상태: 승인 대기(pub-8134930906845147). `docs/adsense-compliance.md` 정본 준수.

### Track 2 — 금융 제휴 / CPA (주력 업사이드, 트래픽 임계 후)
왜 우리에게 맞나 — **우리 도구가 곧 금융 전환 페이지**다:
| 도구 | 자연스러운 제휴 |
|---|---|
| loan-calc · prepayment-penalty | 대출 금리비교 (핀다·토스·카카오페이 등) |
| insurance-calc · unemployment-benefit · foreign-health-insurance | 보험 비교 |
| net-salary · income-tax · freelancer-tax | 카드·계좌개설·세무 서비스 |
| capital-gains-tax · gift-tax · brokerage-fee | 부동산·세무 상담 |

- **방문자당 수익이 AdSense의 10~50배.** 대출 CPA 1건 1~5만 원, 카드 발급 수만 원 수준.
- **검증된 채널**(2026-06 조사):
  - 오픈 제휴망: **링크프라이스**(정식 제휴 다수, 승인 까다로움 — 트래픽 필요), **애드픽**(상대적 오픈, CPA), **텐핑**, **리더스CPA**, 쿠팡파트너스(CPS·저트래픽 OK).
  - 금융 DB형 CPA: **제이커브인터렉티브 · 믹스앤픽스** 같은 대행사, 또는 대출비교 플랫폼 **직접 제휴**.
- **⚠️ 현실 체크**: 좋은 제휴망·금융 CPA는 **승인에 트래픽이 필요**하다(링크프라이스 = "어느 정도 트래픽 있는 채널"). 즉 제휴는 트래픽 병목을 우회하지 못한다 — **트래픽이 커진 뒤의 업그레이드**다.

### Track 3 — 트래픽 (Track 1·2 공통 병목 = 최우선)
- **가장 빠른 성장**: 평균순위 12.4(2페이지)에 걸린 키워드를 1페이지로 밀어 올리기.
  - GSC → 실적 → 쿼리에서 **게재순위 11~20위** 키워드 확인 → 그 페이지의 제목·본문·내부링크 보강 + 색인 요청.
- **권위(백링크)**: GitHub README, 네이버 블로그/티스토리, 커뮤니티 1~2개. "색인 안 됨" 20개를 푸는 진짜 레버.
- **배포**: 네이버 블로그(한국어)·Reddit r/Living_in_Korea·expat 그룹(영어). 진짜 사용자만(AdSense invalid traffic 금지).
- **영어 니치**: 낮은 경쟁 + 높은 RPM + Reddit 배포 용이 → 트래픽·단가 동시 유리.

## 2. 순서와 원칙 (중요)

1. **AdSense 승인 전에는 라이브에 제휴 링크를 올리지 않는다.** 심사 중 광고 밀도·구성 변경은 심사를 복잡하게 만들 수 있다. 승인 후 활성화.
2. **제휴 링크는 "제휴 링크/광고" 공시 필수** (한국 표시광고법 + AdSense 정책). 기만적 배치(내비·다운로드로 오인) 금지. `rel="sponsored"` 부여.
3. **AdSense + 제휴 병행은 공식 허용** (support.google.com/adsense/answer/9728). 단 저품질 페이지 양산 금지 — 우리는 ToolGuide 고유 본문이 있어 thin-affiliate 아님.
4. **제휴 신청은 트래픽 임계 후** (목표: 월 1만 PV+ 또는 일 300+ 방문). 그 전엔 승인 거절 가능.

## 4. 기술 준비 상태

- **OfferSlot 컴포넌트**(`components/seo/OfferSlot.tsx`) + **`lib/offers.ts`**(toolKey→Offer 단일 진실원)가 **휴면 상태로 준비**됨.
  - `OFFERS` 가 비어 있어 **현재 아무것도 렌더되지 않음**(AdSense 심사에 무해).
  - 파트너 링크 확보 시 `lib/offers.ts` 의 `OFFERS` 에 도구별 오퍼를 넣고, 해당 폼의 `ResultShell` 에 `toolKey` 를 전달하면 결과 하단에 **"제휴 링크" 공시와 함께** 자동 노출.
- AdSense 코드(`AdSenseScript`)는 `<head>` 유지. ads.txt·GSC·canonical·www→apex 정리 완료.

---

**한 줄 요약**: 지금 방향은 옳다. AdSense는 바닥 수익으로 깔되, **주력은 금융 제휴/CPA + 영어 니치**로 가고, 그 전제인 **트래픽(2페이지→1페이지 + 배포 + 백링크)** 을 최우선으로 키운다. 제휴 인프라는 준비돼 있으니, 트래픽이 임계에 닿고 AdSense가 승인되면 링크만 꽂으면 된다.
